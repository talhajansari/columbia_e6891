     NAVEGAÇÃO

     Gostaria de poder aproximar (zoom in) /afastar (zoom out) e avançar ao invés de cortar o arquivo de som em seções.
     Isto é perfeitamente possível. O método de cortar em seções foi utilizado no início como uma solução simples para limitações de memória RAM e de velocidade de navegação durante as análises. Embora não seja perfeito, o método funciona, fazendo com que esse assunto ainda não tenha chegado ao topo das lista de prioridades. No entanto, não deve demorar muito.

     O que é aquela tela preta que aparece quando estou abrindo o Sound Ruler?
     Aquela tela mostra mensagens de avisos e erros de código. Quando as coisas não funcionarem quando deveriam, copie o conteúdo desta tela para o seu email de notificação de erros.

     ANÁLISE

     Como faço para apagar as barras verdes quando erro o clique?
     Não há um botão de "apagar". Clique sobre o número da seção ou canto corrente na lista Sections ou Calls. O gráfico será redesenhado, apagando todas as barras verdes.

     É possível fazer o programa plotar o oscilograma do canto sempre com a mesma duração (para gráficos e CCA)?
     Por que o programa excede as barras verdes que eu defino quando faço reconhecimento manual na seção?
     SoundRuler permite controlar quanto som é adicionado ao redor do canto reconhecido na seção. No menu Options, Section a àrea "Plot: time displayed around the call" define quanto som é incorporado no próximo gráfico.
     Para que o programa obedeça sua seleção manual (barras verdes) à risca, defina "centered", "percentage" e 100 %.
     Para que o programa selecione uma quantidade fixa de som, defina "centered", "milliseconds" e especifique uma duração. O programa vai selecionar a duração desejada, centralizando-a entre as barras verdes de seleção manual ou o contorno azul de reconhecimento automático.

     Porque o Sound Ruler não mede a variável xyz?
     Envie um email descrevendo porque a variável xyz é importante para o seu trabalho. A implementação desta medição pode ser muito simples e pode apenas não me ter ocorrido incorporá-la.

     GRÁFICO

     Quero editar minhas figuras arrastando e soltando com o mouse.
     Eu também, mas as bibliotecas do Matlab que a versão executável utiliza ainda não suportam a função arrastar e soltar. No entanto, as bibliotecas do próprio Matlab suportam, então se você tem o Matlab, é possível utilizar a função arrastar e soltar com a versão script do Sound Ruler.

     Porque o Sound Ruler não salva os meus espectrogramas como Adobe Illustrator ou Windows metafiles?
     (Parece que agora já está salvando, ou pelo menos copiando e colando, mas precisamos verificar mais alguns sistemas.)
     Esses são formatos vetoriais, o que significa que a imagem é definida como uma série de objetos geométricos num mapa de coordenadas. Espectrogramas são gerados como bitmaps, onde a cor de cada pixel é definida ponto a ponto. É bastante fácil converter imagens vetoriais em bitmaps, mas não o oposto, especialmente quando a imagem apresenta gradientes complexos de cinza.

     LANÇAMENTOS

     Quais são os tipos disponíveis de Sound Ruler?
     O Sound Ruler é lançado como um executável binário ou scripts do Matlab. As versões executável são mais rápidas e leves do que os scripts e não requerem o Matlab. Os scripts são mais fáceis de modificar e possuem algumas funções adicionais para edição de gráficos. Tanto executáveis como scripts estão disponíveis para Windows e para Linux. A versão Mac ainda não foi lançada. Os arquivos para download são:
     soundruler-x.x.x.windows.zip: binários da versão executável para Windows .
     soundruler-x.x.x.windows.scripts.zip: scripts para uso dentro do Matlab 6.x. Não devem necessitar de muitos ajustes para rodar em versões antigas do Matlab ou outros sistemas operacionais.
     soundruler-x.x.x.linux.rpm: binários da versão auto-executável para Linux. Devem funcionar em sistemas unix em geral.
     soundruler-x.x.x.linux.tar.gz: binários para Linux com scripts de instalação Makefile e spec.
     soundruler-x.x.x.linux.scripts.tar.gz: scripts para uso dentro do Matlab 6.x

     Onde estão os arquivos de código? Qual é a linguagem usada para escrever o Sound Ruler?
     O Sound Ruler é desenvolvido em scripts M-file . Estes são implementados com Octave (aberto, gratuito) e Matlab (comercial). Os arquivos de código estão nas versões script.

     Ondes estão os binários para Mac OS X?
     Eles não foram lançados ainda. Se você tem Matlab para OS X e Matlab, é possível utilizar a versão script M-file (pequenos ajustes podem ser necessários).

     OUTROS

     Alguns comandos que aparecem na documentação estão faltando na minha versão do SoundRuler.
     Nós estamos lidando com quatro versões de programa, cada uma rodando em bibliotecas diferentes. No momento não é possivel fazer com que todos os comandos estejam presentes e se comportem da mesma forma nas quatro versões.

     O Sound Ruler roda em Octave?
     A interface gráfica que o Sound Ruler utiliza ainda não está disponível em Octave. Mas é possível adaptar a sua funcionalidade para o console e usar o GnuPlot para gerar os gráficos. Confira como foi feito no Listener, um sistema de monitoramento acústico automatizado.

     O Sound Ruler pode co-existir com as instalações de Matlab?
    Se a versão de Matlab utilizar as mesmas bibliotecas enviadas com o Sound Ruler, não é preciso instalar as bibliotecas. Se elas não forem da mesma versão, no Windows é preciso se certificar de que a chamada para a pasta do SoundRuler no caminho aparece antes da chamada para a pasta do Matlab e ambos devem funcionar perfeitamente.

     SOLUÇÕES PARA PROBLEMAS

     "A required .dll file, libmmfile.dll, was not found".
     "Sorry, the configuration of the path failed".
     Por algum motivo, a instalação não conseguiu configurar o caminho. Basta informar ao Windows aonde as bibliotecas matematicas estao.

     No Windows98 ou Windows95: encontre o arquivo c:\autoexec.bat. Se ele não existir, crie um. Edite ou adicione a linha que começa com "SET PATH=" e inclua a chamada C:\MyFolder\soundruler\bin\win32; substituindo C e MyFolder pelo drive e pasta aonde você instalou o Sound Ruler. (veja PARA TODOS abaixo).

    No Windows2000/NT: vá em Iniciar - Configurações - Painel de Controle - Sistema. Uma janela vai abrir com várias abas no topo. Escolha Avançado e ao fundo, clique em Variáveis Ambientais. Uma janela vai se abrir com duas caixas. A de baixo, Variáveis Ambientais, tem um ítem chamado "Caminho". Clique em "Caminho" e no botão Editar para adicionar o caminho para as bibliotecas matemáticas. Se o botão Editar não estiver disponível, seu usuário não tem poderes administrativos. Neste caso, faça login como Administrador. Ao clicar no botão Editar, uma janela se abre com dois campos. O de baixo "Valor da variável" é aonde se deve adicionar c:\arquivos de programas\soundruler\bin\win32, modificando c:\arquivos de programas\soundruler pelo caminho ande o SoundRuler foi instalado.

    PARA TODOS: Se você tem o Matlab instalado, coloque a nova chamada no caminho ANTES da chamada para a pasta do Matlab. Isto não deve afetar o Matlab. Agora reinicialize o computador e SoundRuler deve funcionar. Caso não funcione, pode haver um erro de digitação no caminho. Este pode ser verificado abrindo uma janela do DOS (Iniciar - Executar, "command" ou "cmd") e entrando o comando "path". Ele deve mostrar algo como:
          PATH=C:\Windows\system32;c:\program files\soundruler\bin\win32;

     Meus arquivos de configuração pararam de funcionar.
     Os arquivos de configuração são sensíveis à taxa de amostragem. Para utilizá-los com uma nova taxa de amostragem, talvez seja necessário modificar todas as configurações que são baseadas no número de amostras. Talvez eu invente um conversor automático...

     "The ordinal #### could not be located in the dynamic-link library dforrt.dll".
     Este é um problema com a instalação das bibliotecas do Matlab. Aqui está a solução da Mathworks: localize o dforrt.dll ou deformd.dll no diretório do seu sistema Windows e substitua-os pelos arquivos correspondentes do diretório <soundruler>\bin\win32, onde <soundruler> é o nome do diretório de instalação do Sound Ruler.

     O som nas versões para Linux está ruim ou ausente.
     A implementação de som nas bibliotecas para Linux ainda está precária. A qualidade do som parece depender bastante tanto do hardware quanto do software do sistema, de modo que pode valer a pena tentar outra máquina.
